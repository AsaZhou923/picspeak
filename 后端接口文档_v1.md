# 后端接口文档（v1）

## 1. 基础信息

- Base URL: `/api/v1`
- 鉴权: `Authorization: Bearer <jwt>`（HS256，必须包含 `sub` 声明，可选校验 `iss/aud`）
- 登录体系: Google 登录换取 JWT；未登录用户可先申请游客 JWT
- 用户等级: `guest` / `free` / `pro`；限流倍率分别为 `0.5x / 1x / 2x`
- 自动游客: 未携带 Authorization 时，服务端自动创建游客并通过 Cookie 返回 `ps_guest_token`
- 每日评图额度（UTC 日切）：`3 / 6 / 12` 次
- Content-Type: `application/json`
- 时间字段: ISO8601 UTC
- 幂等: 创建类接口支持 `Idempotency-Key`

通用错误体：

```json
{
  "error": {
    "code": "INVALID_ARGUMENT",
    "message": "exif_data is invalid",
    "request_id": "req_123"
  }
}
```

## 2. 状态机约定

- `photo.status`: `UPLOADING | READY | REJECTED`
- `task/review.status`: `PENDING | RUNNING | SUCCEEDED | FAILED | EXPIRED`

## 3. 接口列表

### 3.0 鉴权

`POST /auth/guest`

响应：

```json
{
  "access_token": "<jwt>",
  "token_type": "bearer",
  "user_id": "gst_xxx",
  "plan": "guest"
}
```

`POST /auth/google/login`

请求：

```json
{
  "id_token": "<google-id-token>"
}
```

响应：

```json
{
  "access_token": "<jwt>",
  "token_type": "bearer",
  "user_id": "google-sub",
  "plan": "free"
}
```


`GET /auth/google/callback?code=...`

说明：

- 适用于无前端 OAuth SDK 时，后端直接接收 `code`，向 Google 换取 token 并完成登录。
- 需要配置 `GOOGLE_OAUTH_CLIENT_ID` / `GOOGLE_OAUTH_CLIENT_SECRET` / `GOOGLE_OAUTH_REDIRECT_URI`。

### 3.1 创建上传签名

`POST /uploads/presign`

请求：

```json
{
  "filename": "DSC001.jpg",
  "content_type": "image/jpeg",
  "size_bytes": 5242880,
  "sha256": "optional-file-sha256"
}
```

响应：

```json
{
  "upload_id": "upl_xxx",
  "object_key": "user_123/2026/03/xxx.jpg",
  "put_url": "https://...",
  "headers": {
    "Content-Type": "image/jpeg"
  },
  "expires_at": "2026-03-01T12:00:00Z"
}
```

### 3.2 确认上传并落库照片

`POST /photos`

请求：

```json
{
  "upload_id": "upl_xxx",
  "exif_data": {
    "iso": 200,
    "shutter": "1/200",
    "aperture": "f/2.8",
    "focal_length_mm": 35
  },
  "client_meta": {
    "width": 2048,
    "height": 1365
  }
}
```

响应：

```json
{
  "photo_id": "pho_xxx",
  "photo_url": "s3://bucket/user_123/2026/03/xxx.jpg",
  "status": "READY"
}
```

### 3.3 发起 AI 点评任务

`POST /reviews`

请求：

```json
{
  "photo_id": "pho_xxx",
  "mode": "flash",
  "async": true,
  "idempotency_key": "client-unique-key"
}
```

异步响应：

```json
{
  "task_id": "tsk_xxx",
  "status": "PENDING",
  "estimated_seconds": 12
}
```

同步响应：

```json
{
  "review_id": "rev_xxx",
  "status": "SUCCEEDED",
  "result": {
    "schema_version": "1.0",
    "scores": {
      "composition": 8,
      "lighting": 7,
      "color": 9,
      "story": 6,
      "technical": 8
    },
    "advantage": "构图有创意，颜色控制较好。",
    "critique": "前景元素略杂，主体分离度不足。",
    "suggestions": "可裁切下方 10%，并降低高光 20%。"
  }
}
```

### 3.4 查询任务状态

```json
  "status": "PENDING",

  "review_id": null,
  "error": null

```

`GET /ws/tasks/{task_id}`

- `task.progress`
- `task.succeeded`
- `task.failed`

## 4. 服务端强约束（实现要求）

- `/reviews` 仅接受 `photo_id`，不直接信任前端 `photo_url`
- 所有创建类接口支持幂等键
- AI 结果必须通过 JSON Schema 校验并存 `schema_version`
- 异步任务必须具备重试、超时、死信队列
- 上传前后均执行文件与对象归属校验
