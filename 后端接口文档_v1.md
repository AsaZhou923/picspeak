# 后端接口文档（v1）

本文档以当前代码实现为准，覆盖 `app/api/routes.py` 中已落地接口。

## 1. 基础信息

- Base URL：`/api/v1`
- Content-Type：`application/json`
- 时间格式：ISO8601
- 鉴权方式：`Authorization: Bearer <access_token>`
- 游客模式：未携带 `Authorization` 且未携带游客 Cookie 时，服务端会自动创建游客用户，并通过 Cookie `ps_guest_token` 返回令牌
- 错误响应：统一为 `{"error":{"code","message","request_id"}}`

## 2. 用户与权限模型

### 用户等级

- `guest`
- `free`
- `pro`

### 权限约束

- `guest` 用户不能使用 `pro` 点评模式
- 所有照片、点评、任务查询都只允许访问当前用户自己的资源

### 每日额度规则

服务端基于 `DEFAULT_DAILY_QUOTA` 动态计算：

- `guest`：基线一半，至少 1 次
- `free`：基线额度
- `pro`：基线两倍

## 3. 状态字段

### Photo.status

- `UPLOADING`
- `READY`
- `REJECTED`

### Task.status

- `PENDING`
- `RUNNING`
- `SUCCEEDED`
- `FAILED`
- `EXPIRED`
- `DEAD_LETTER`

### Review.status

- `SUCCEEDED`
- `FAILED`

## 4. 鉴权接口

### 4.1 获取游客令牌

`POST /auth/guest`

说明：

- 创建游客用户
- 返回 JWT
- 同时写入 Cookie `ps_guest_token`
- 响应头包含 `X-Guest-Access-Token`

响应示例：

```json
{
  "access_token": "<jwt>",
  "token_type": "bearer",
  "user_id": "gst_xxx",
  "plan": "guest"
}
```

### 4.2 Google ID Token 登录

`POST /auth/google/login`

请求体：

```json
{
  "id_token": "<google-id-token>"
}
```

响应示例：

```json
{
  "access_token": "<jwt>",
  "token_type": "bearer",
  "user_id": "google_sub",
  "plan": "free"
}
```

说明：

- 服务端会校验 `email_verified`
- 如果配置了 `GOOGLE_OAUTH_CLIENT_ID`，还会校验 `aud`
- 首次登录会创建用户，默认 `plan=free`

### 4.3 Google OAuth 回调登录

`GET /auth/google/callback?code=xxx`

说明：

- 后端使用 `code` 向 Google 换取 `id_token`
- 登录完成后，302 跳转到 `${FRONTEND_ORIGIN}/auth/callback/google`
- 成功时把 `access_token`、`token_type`、`user_id`、`plan` 追加到 query string
- 失败时把错误信息放到 `error` 参数

依赖配置：

- `GOOGLE_OAUTH_CLIENT_ID`
- `GOOGLE_OAUTH_CLIENT_SECRET`
- `GOOGLE_OAUTH_REDIRECT_URI`
- `FRONTEND_ORIGIN`

## 5. 上传接口

### 5.1 获取上传预签名

`POST /uploads/presign`

请求体：

```json
{
  "filename": "DSC001.jpg",
  "content_type": "image/jpeg",
  "size_bytes": 5242880,
  "sha256": "optional-file-sha256"
}
```

请求约束：

- `content_type` 仅支持：
  - `image/jpeg`
  - `image/png`
  - `image/webp`
- `size_bytes` 必须大于 0
- `size_bytes` 不能超过 `MAX_UPLOAD_BYTES`

响应示例：

```json
{
  "upload_id": "eyJhbGciOi...",
  "object_key": "user_gst_xxx/2026/03/obj_xxx.jpg",
  "put_url": "https://example.r2.cloudflarestorage.com/...",
  "headers": {
    "Content-Type": "image/jpeg"
  },
  "expires_at": "2026-03-07T05:10:00Z"
}
```

说明：

- `upload_id` 是服务端签名令牌，不是普通业务 ID
- 客户端应使用返回的 `put_url + headers` 直接 `PUT` 到对象存储
- 预签名有效期为 10 分钟

### 5.2 确认上传并创建照片记录

`POST /photos`

请求体：

```json
{
  "upload_id": "eyJhbGciOi...",
  "exif_data": {
    "iso": 200,
    "shutter": "1/200",
    "aperture": "f/2.8",
    "focal_length_mm": 35
  },
  "client_meta": {
    "width": 2048,
    "height": 1365
  }
}
```

请求约束：

- `upload_id` 必须是服务端生成且未过期的签名令牌
- `upload_id` 中绑定的用户必须和当前登录用户一致
- `client_meta.width`、`client_meta.height` 如果传入，必须大于 0

响应示例：

```json
{
  "photo_id": "pho_xxx",
  "photo_url": "https://cdn.example.com/user_gst_xxx/2026/03/obj_xxx.jpg",
  "status": "READY"
}
```

## 6. 点评接口

### 6.1 创建点评任务 / 同步点评

`POST /reviews`

请求体：

```json
{
  "photo_id": "pho_xxx",
  "mode": "flash",
  "async": true,
  "idempotency_key": "client-unique-key",
  "locale": "zh"
}
```

字段说明：

- `mode`：`flash` 或 `pro`
- `async`：默认 `true`
- `idempotency_key`：可选，也可通过请求头 `Idempotency-Key` 传入
- `locale`：`zh` / `en` / `ja`

请求约束：

- 当前用户状态必须为 `active`
- `guest` 用户不能使用 `pro`
- `photo_id` 必须属于当前用户
- 照片状态必须为 `READY`
- 服务端会校验每日额度

#### 异步响应示例

```json
{
  "task_id": "tsk_xxx",
  "status": "PENDING",
  "estimated_seconds": 12
}
```

#### 同步响应示例

```json
{
  "review_id": "rev_xxx",
  "status": "SUCCEEDED",
  "result": {
    "schema_version": "1.0",
    "scores": {
      "composition": 8,
      "lighting": 7,
      "color": 9,
      "story": 6,
      "technical": 8
    },
    "final_score": 7.6,
    "advantage": "构图有创意，颜色控制较好。",
    "critique": "前景元素略杂，主体分离度不足。",
    "suggestions": "可裁切下方 10%，并降低高光 20%。"
  }
}
```

说明：

- 同步模式会直接调用 AI 并返回结果
- 异步模式会写入 `review_tasks`，由进程内 worker 处理
- 若开启 `IMAGE_AUDIT_ENABLED=true`，点评前会先进行图片审核
- 幂等键命中时，服务端会直接返回上次缓存结果

### 6.2 查询任务状态

`GET /tasks/{task_id}`

响应示例：

```json
{
  "task_id": "tsk_xxx",
  "status": "RUNNING",
  "progress": 70,
  "review_id": null,
  "error": null
}
```

失败示例：

```json
{
  "task_id": "tsk_xxx",
  "status": "FAILED",
  "progress": 100,
  "review_id": null,
  "error": {
    "code": "AI_CALL_FAILED",
    "message": "..."
  }
}
```

说明：

- 仅可查询当前用户自己的任务
- 成功后 `review_id` 会被填充
- 可返回 `DEAD_LETTER`，表示任务已重试耗尽并进入死信队列

### 6.3 WebSocket 任务推送

`GET /ws/tasks/{task_id}`（WebSocket）

连接方式：

- query 参数携带 `access_token`
- 或依赖游客 Cookie `ps_guest_token`

消息示例：

```json
{
  "type": "task.update",
  "task": {
    "task_id": "tsk_xxx",
    "status": "RUNNING",
    "progress": 70,
    "review_id": null,
    "error": null
  },
  "event": {
    "event_type": "AI_REVIEW_STARTED",
    "message": "Running AI review",
    "created_at": "2026-03-07T05:20:00Z"
  }
}
```

说明：

- 服务端会按任务状态变化推送最新快照
- 任务完成、失败、超时或进入死信后，连接会关闭
- WebSocket 推送基于数据库任务状态和事件日志，不依赖 API 进程内内存

## 7. 历史记录接口

### 7.1 查询当前用户点评历史

`GET /me/reviews?limit=20&cursor=2026-03-07T05:20:00+00:00`

查询参数：

- `limit`：默认 20，范围 `1-100`
- `cursor`：上一页最后一条记录的 `created_at`

响应示例：

```json
{
  "items": [
    {
      "review_id": "rev_1",
      "photo_id": "pho_1",
      "photo_url": "https://cdn.example.com/user_xxx/2026/03/obj_xxx.jpg",
      "mode": "flash",
      "status": "SUCCEEDED",
      "final_score": 7.6,
      "created_at": "2026-03-07T05:20:00Z"
    }
  ],
  "next_cursor": null
}
```

`GET /photos/{photo_id}/reviews?limit=20&cursor=2026-03-07T05:20:00+00:00`

查询参数：

- `limit`：默认 20，范围 `1-100`
- `cursor`：上一页最后一条记录的 `created_at`

响应示例：

```json
{
  "items": [
    {
      "review_id": "rev_1",
      "mode": "flash",
      "status": "SUCCEEDED"
    }
  ],
  "next_cursor": null
}
```

说明：

- 仅可查询当前用户自己的照片

## 10. 常见错误

当前项目统一返回以下错误结构：

```json
{
  "error": {
    "code": "PHOTO_NOT_FOUND",
    "message": "Photo not found",
    "request_id": "req_xxx"
  }
}
```

常见错误场景：

- `400 Bad Request`
  - 不支持的 `content_type`
  - 文件过大
  - `photo` 状态不可点评
  - 图片审核未通过
  - 游标格式错误
- `401 Unauthorized`
  - 缺少或无效的访问令牌
  - Google token 校验失败
- `403 Forbidden`
  - 上传归属不一致
  - `guest` 用户使用 `pro`
  - 用户状态不是 `active`
- `404 Not Found`
  - `photo_id` / `task_id` / `review_id` 不存在或不属于当前用户
- `409 Conflict`
  - 重复照片
  - 幂等任务冲突
- `500 Internal Server Error`
  - 对象存储预签名生成失败
  - Google OAuth 回调未配置
- `502 Bad Gateway`
  - Google token 校验失败
  - AI 调用失败
  - 图片审核服务失败

## 11. 任务执行架构说明

当前任务系统已支持：

- 数据库队列
- 独立 worker 进程
- 指数退避重试
- 死信状态 `DEAD_LETTER`
- `review_task_events` 任务事件编排日志

默认仍可通过 `RUN_EMBEDDED_WORKER=true` 在 API 进程内启用嵌入式 worker，生产建议改为独立进程运行 `python -m app.worker_main`。
