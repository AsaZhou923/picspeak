# PicSpeak 全流程测试文档（游客 + Google 登录）

本文档覆盖三条主链路：

1. 游客“零操作”进入即评图（自动游客）
2. Google `id_token` 登录换业务 JWT
3. Google `code` 回调登录（`GET /auth/google/callback`）

---

## 0. 准备

- 启动服务：

```bash
uvicorn app.main:app --reload --port 8000
```

- 先确认健康检查：

```bash
curl http://localhost:8000/healthz
```

期望：`{"status":"ok"}`。

---

## 1. 游客零操作链路（无需先调用 /auth/guest）

> 目标：不带 Authorization 直接访问业务接口，系统自动创建游客并回传 Cookie。\
> 说明：由于上传/评图有先后依赖，建议先测 `/me/usage` 验证自动游客是否生效。

### 1.1 不带 token 调用 `/me/usage`

```bash
curl -i -c cookie.txt 'http://localhost:8000/api/v1/me/usage'
```

验证点：

- 响应头含 `Set-Cookie: ps_guest_token=...`
- 响应体 `plan` 为 `guest`
- `quota.daily_total` 为 `3`

### 1.2 使用 Cookie 再次调用 `/me/usage`

```bash
curl -i -b cookie.txt 'http://localhost:8000/api/v1/me/usage'
```

验证点：

- 不再新建游客，`plan` 仍是 `guest`
- 限流字段里可看到剩余次数递减

### 1.3 游客限流压测（两层保障）

```bash
for i in $(seq 1 12); do
  curl -s -o /dev/null -w "%{http_code}\n" -b cookie.txt 'http://localhost:8000/api/v1/me/usage'
done
```

预期：会出现 `429`，可能来自以下任一层：

- 基线 * plan 倍率（guest 0.5x）
- `guest_ip_rate_limit_per_minute`（默认 8/min）
- `guest_burst_limit_per_10s`（默认 3/10s）

## 2. Google `id_token` 登录链路

### 2.1 配置后端

`.env` 至少包含：

```env
OAUTH_JWT_SECRET=replace-with-strong-secret
GOOGLE_OAUTH_CLIENT_ID=xxx.apps.googleusercontent.com
```

### 2.2 前端/调试工具拿到 Google `id_token`

拿到后执行：

```bash
curl -X POST 'http://localhost:8000/api/v1/auth/google/login' \
  -H 'Content-Type: application/json' \
  -d '{"id_token":"<google-id-token>"}'
```

验证点：

- 返回 `access_token`
- 返回 `plan=free`
- 用该 token 调 `/api/v1/me/usage` 时 `plan=free` 且 `daily_total=6`

## 3. Google `code` 回调链路（新增）

> 适合：你没有前端 SDK，希望后端直接处理 OAuth code。

### 3.1 后端额外配置

`.env` 补齐：

```env
GOOGLE_OAUTH_CLIENT_ID=xxx.apps.googleusercontent.com
GOOGLE_OAUTH_CLIENT_SECRET=xxx
GOOGLE_OAUTH_REDIRECT_URI=http://localhost:8000/api/v1/auth/google/callback
```

并确保 Google Cloud Console 的 redirect URI 与上面完全一致。

### 3.2 浏览器获取 `code`

访问 Google 授权 URL（示例）：

```text
https://accounts.google.com/o/oauth2/v2/auth?client_id=<CLIENT_ID>&redirect_uri=<REDIRECT_URI>&response_type=code&scope=openid%20email%20profile&access_type=online&prompt=consent
```

登录同意后会跳到回调地址，URL 里会带 `?code=...`。

### 3.3 调回调接口

```bash
curl 'http://localhost:8000/api/v1/auth/google/callback?code=<google-code>'
```

验证点：

- 返回业务 `access_token`
- 返回用户 `plan=free`
- 该 token 可访问所有受保护业务接口

## 4. 评图主链路回归（适用于游客和 Google 用户）

1. `POST /api/v1/uploads/presign`
2. 前端/脚本 PUT 上传对象存储
3. `POST /api/v1/photos`
4. `POST /api/v1/reviews`
5. `GET /api/v1/tasks/{task_id}` 或 `GET /api/v1/reviews/{review_id}`

检查点：

- 游客可完整走通（无需提前登录）
- 达到日限额后返回 429
- Google 用户额度与限流高于游客

---

## 5. 常见失败与定位

- `401 Invalid access token`：检查 Bearer token 是否过期/签名配置是否一致。
- `401 Invalid Google token audience`：`GOOGLE_OAUTH_CLIENT_ID` 与 token `aud` 不匹配。
- `500 Google OAuth callback is not configured`：缺少 `GOOGLE_OAUTH_CLIENT_SECRET` 或 `GOOGLE_OAUTH_REDIRECT_URI`。
- `429 Rate limit exceeded (...)`：命中分层限流，建议先放缓请求节奏再重试。
- `429 Daily quota exceeded`：当天评图额度已用尽，次日 UTC 自动重置。
