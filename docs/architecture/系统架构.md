### 1. 系统架构图 (System Architecture)

- 前端：Next.js (App Router) + React Hooks + Web Worker 图片预处理
- API 层：FastAPI（鉴权、配额、限流、幂等）
- 任务层：Queue Worker（异步点评、重试、死信、超时）
- AI 层：Vision LLM（Flash/Pro 分层）+ 结果 Schema 校验
- 存储层：R2（私有对象）+ PostgreSQL（业务数据）

------

### 2. 核心组件详解

#### A. 前端层 (The Entry)

- **框架**：Next.js (App Router)。SSR 提升 SEO，Client Components 处理上传与预览。
- **图片预处理**：上传前使用 Web Worker + Canvas 压缩（建议长边 2048px）。
- **状态管理**：React Hooks（useState/useEffect）+ WebSocket/轮询 管理点评任务状态。
- **调用约束**：前端仅上送 `photo_id` 触发点评，不直接上送可任意伪造的 `photo_url`。

#### B. 后端服务层 (The Logic)

- **框架**：FastAPI（Python）。
- **核心模块**：
  1. **Upload Guard**：签名上传、MIME/尺寸/像素校验、对象归属校验。
  2. **Metadata Extractor**：提取并标准化 EXIF（ISO、快门、光圈、焦距等）。
  3. **Prompt Orchestrator**：将图片上下文 + EXIF + 用户模式（flash/pro）组装为结构化 Prompt。
  4. **Result Parser**：强制 AI 返回 JSON，并按 Schema 校验后入库。
  5. **Quota & RateLimit**：按 `IP + user_id + endpoint + plan` 多维限流。
  6. **Idempotency Manager**：创建类接口支持 `Idempotency-Key`，避免重复扣费。

#### C. 存储与数据层 (The Memory)

- **对象存储**：Cloudflare R2
- **上传策略**：服务端签发短时效 Presigned URL；回调/确认时校验对象 key 与所属用户绑定。
- **数据库**：PostgreSQL。
- **建议表结构**：
  - `users`：用户信息、套餐、配额。
  - `photos`：对象 key、EXIF、状态、所有者。
  - `review_tasks`：任务状态机、重试次数、错误信息、幂等键。
  - `reviews`：模型结果 JSON、schema_version、耗时、token 成本。

#### D. AI 智能层 (The Intelligence)

- **模型策略**：
  - `flash`：快速反馈，低成本。
  - `pro`：深度点评，高价值场景。
- **结果协议**：固定 JSON Schema（含 `schema_version`），字段缺失/类型错误时进入兜底流程。
- **稳定性策略**：解析失败重试（有限次），失败后返回可读错误并记录观测指标。

#### E. 异步任务层 (The Queue)

- **触发条件**：点评请求默认异步化（可为小图/低延迟场景保留同步开关）。
- **任务状态机**：`PENDING -> RUNNING -> SUCCEEDED | FAILED | EXPIRED`。
- **可靠性机制**：
  - 超时控制（单任务超时、总超时）。
  - 指数退避重试（含最大次数）。
  - 死信队列（DLQ）用于人工排查。
  - 幂等消费避免重复执行。

------

### 3. 关键业务流程 (Data Flow)

1. **上传阶段**
   - 前端请求 `POST /uploads/presign`。
   - 客户端直传到 R2/S3。
   - 前端调用 `POST /photos` 确认上传并提交 EXIF。
   - 后端校验对象归属、文件类型与安全策略，写入 `photos`。

2. **触发点评**
   - 前端调用 `POST /reviews`（仅传 `photo_id` + `mode` + `Idempotency-Key`）。
   - 后端校验配额与限流后写入 `review_tasks`，返回 `task_id`。

3. **异步处理**
   - Worker 拉取任务，完成 NSFW 预检与 Prompt 编排。
   - 调用 Vision LLM。
   - 对响应进行 JSON Schema 校验。
   - 成功：写入 `reviews` 并更新任务为 `SUCCEEDED`。
   - 失败：按重试策略处理，最终进入 `FAILED` 或 `DLQ`。

4. **结果获取**
   - 前端轮询 `GET /tasks/{task_id}` 或订阅 WebSocket。
   - 任务成功后调用 `GET /reviews/{review_id}` 获取点评结果。

------

### 4. 安全与风控设计

- **输入可信边界**：后端不信任前端 `photo_url` 与原始 EXIF；必须二次校验。
- **文件安全**：限制 MIME、大小、像素，防止解码炸弹与伪装文件。
- **内容安全**：LLM 前置 NSFW 检测，违规内容直接拦截。
- **访问控制**：对象存储默认私有，短时 URL，最小权限访问。
- **防刷策略**：全量用户限流 + 日配额 + 异常行为告警。

------

### 5. 性能与成本优化

- **冷热分离**：点评结果入库缓存，历史查看直接读 DB，不重复消耗 Token。
- **分层模型**：优先 `flash`，按套餐或用户主动选择升级 `pro`。
- **图像预处理**：前端压缩减轻带宽与推理输入成本。
- **可观测性**：记录每次点评耗时、重试次数、token 消耗与失败原因。

------

### 6. 交付建议

- 优先实现：上传闭环（签名上传+确认）-> 异步任务 -> 结果 Schema 校验 -> 基础限流。
- 二期增强：WebSocket 推送、精细化配额策略、运营看板与审计报表。
