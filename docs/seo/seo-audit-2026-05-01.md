# PicSpeak SEO / GEO 全量审计报告

审计日期：2026-05-01  
审计范围：`frontend` Next.js 应用、公开路由、metadata、sitemap、robots、llms.txt、结构化数据、内容资产与关键产品入口  
站点：<https://picspeak.art>  
项目类型：AI 摄影点评 SaaS + GPT Image 2 图像参考生成工具

## 1. 总览

PicSpeak 的 SEO 基础已经比较成熟：Next.js App Router 输出 SSR 页面，公开首页、多语言首页、博客、更新记录、影像长廊、示例评图均已有 metadata 或结构化数据；`/llms.txt` 与 `/.well-known/llms.txt` 也已经存在，适合面向 AI 搜索和引用系统建立上下文。

本轮审计发现的主要问题不是“缺基础 SEO”，而是新功能上线后的 SEO 信号没有完全同步：

- `/generate` AI Create 页面是新的重要公开功能，但没有独立 metadata，也不在 sitemap 中。
- `/generation-tasks/[taskId]` 和 `/generations/[generationId]` 是用户生成流程和私有结果页，但继承了根站默认可索引 metadata，存在被索引风险。
- `/gallery`、`/affiliate` 和公开 demo review 的 hreflang 指向了 `/zh`、`/ja` 这类非等价页面，会给搜索引擎传递错误语言替代关系。
- sitemap 的语言 alternates 没有总是包含 self/x-default，且公开功能页优先级分配没有覆盖 AI Create。
- `frontend/public/llms.txt` 与动态 `frontend/src/lib/llms.ts` 内容不一致，价格口径和新生成图能力不统一。
- 根站 `siteConfig` 仍以“AI 摄影点评”为唯一描述，未覆盖 GPT Image 2 生成参考图这一新增产品面。

## 3. 关键发现

### P0：用户生成流程页需要 noindex

路径：

- `frontend/src/app/generation-tasks/[taskId]/page.tsx`
- `frontend/src/app/generations/[generationId]/page.tsx`

问题：这些页面是登录态相关的生成流程/结果页，当前没有 route-level metadata，会继承根布局的 indexable robots。即使实际数据需要鉴权，搜索引擎仍可能抓到空状态、错误状态或包含私有语义的页面。

建议：新增对应 layout，设置 `NO_INDEX_ROBOTS`，并提供明确但不面向排名的 title/description。

### P1：`/generate` 缺少独立 SEO 入口

路径：

- `frontend/src/app/generate/page.tsx`
- `frontend/src/app/sitemap.ts`
- `frontend/src/lib/site.ts`
- `frontend/src/lib/llms.ts`

问题：AI Create / GPT Image 2 生成图已经是产品主功能，但页面 metadata 仍继承“AI photo critique”根站描述，sitemap 也没有收录 `/generate`。这会削弱“AI image generation / visual reference / GPT Image 2 prompt library”相关查询的入口信号。

建议：为 `/generate` 新增专属 metadata、Open Graph、Twitter card、canonical；加入 sitemap；同步 llms.txt 和 siteConfig 的产品描述。

### P1：非等价页面 hreflang 指向错误

路径：

- `frontend/src/app/gallery/layout.tsx`
- `frontend/src/app/affiliate/page.tsx`
- `frontend/src/app/reviews/[reviewId]/layout.tsx`
- `frontend/src/app/sitemap.ts`

问题：`/gallery`、`/affiliate` 和 demo review 不是 `/zh` 或 `/ja` 的语言变体，但 metadata/sitemap 里把它们标成语言 alternate。搜索引擎可能把不同意图页面误解为同一内容的语言版本。

建议：这些混合语言/单 URL 页面只保留 canonical 和 x-default，不再指向 locale home。真正有等价语言版本的页面（首页、博客、更新页）继续保留完整 hreflang。

### P1：llms.txt 内容口径不一致

路径：

- `frontend/src/lib/llms.ts`
- `frontend/public/llms.txt`

问题：动态 llms 内容已经包括较完整站点结构，但未覆盖 AI Create / GPT Image 2 生成图；静态 public 版本仍写着 `$2.99/month` 促销价，与当前 i18n 的 `$3.99/month`、中文 `$1.99 / 30 天` 口径不一致。

建议：统一动态与静态 llms 内容，补充 AI Create、图像 credits、价格地区差异和更准确的 citation guidance。

### P2：sitemap alternates 可读性和一致性可提升

路径：

- `frontend/src/app/sitemap.ts`

问题：locale 页面和博客页面的 `alternates.languages` 有时不包含当前页面自身；root 也缺少 `x-default`。虽然不一定阻断索引，但一致性不足，不利于后续维护。

建议：集中生成 locale alternates，所有 locale 等价页面包含 `zh-CN`、`en`、`ja`、`x-default`。

## 6. 本轮完成记录

已落地代码层优化：

- 新增 `/generate` route metadata 和 SoftwareApplication JSON-LD，覆盖 AI image generator、GPT Image 2 prompts、visual reference generator 等查询意图。
- 新增 `/generation-tasks/[taskId]` 与 `/generations/[generationId]` layout，统一使用 `NO_INDEX_ROBOTS`。
- 修正 `/gallery`、`/affiliate`、demo review 的 alternates，只保留 canonical / x-default，避免误指到 locale home。
- 重构 sitemap 的 locale alternates 生成逻辑，补齐 self 与 x-default，并新增 `/generate`、`/blog`、根博客文章 canonical 入口。
- 更新 robots 输出，明确允许 Googlebot、Bingbot、GPTBot、ChatGPT-User、OAI-SearchBot、ClaudeBot、PerplexityBot、Google-Extended、CCBot。
- 同步 `frontend/src/lib/llms.ts` 和 `frontend/public/llms.txt`，补齐 AI Create、GPT Image 2、credits、地区价格差异和 citation guidance。
- 更新 `siteConfig` 的 title、description、keywords，使根站产品描述覆盖“AI 摄影点评 + GPT Image 2 视觉参考生成”双产品面。

- 构建产物抽查：`sitemap.xml` 已包含 `/generate`；`/gallery`、`/affiliate` 的 hreflang 只输出 `x-default`；`robots.txt` 已输出主要搜索与 AI crawler allow 规则。
